# Отчёт по итогам тестирования #

Выполнено тестирование комплексного сервиса, взаимодействующего с СУБД и API Банка. 
Тестирование проводилось на странице *"Путешествие дня"*, формы *"Купить"* и *"Купить в кредит"*

Всего выполнено 34 автоматизированных тест-кейсов:
27 тестов успешно 79.41%
7 тестов не успешно 20.59%

## Найденные дефекты ##
1. Оплаты картой со статусом DECLINED проходят успешно. Должно быть оповещение об отказе в проведении операции
2. В поле "Владелец" не найден элемент с сообщением об ошибке

## Общие рекомендации ##
1. Пересмотреть логику валидации, возвращать явный статус ошибки при нарушении правил
2. Устранить баги, которые заведены https://github.com/vicfadeewa/summary-qamid-q118/issues
3. Блокировать кнопку "Продолжить", пока все поля не будут заполнены. Она должна быть изначально неактивной
4. Осуществлять мгновенную валидацию данных. Фильтровать недопустимые символы при вводе, проверять корректность заполнения при каждом изменении и потере фокуса
