## Перечень используемых инструментов ##

1. **IntelliJ IDEA** - в проекте для написания и отладки кода, управления зависимостями, запуска тестов, работа с Git.
2. **Java** - для реализация тестовых сценариев, интеграция с Selenide, JUnit, работа с API и БД
3. **Selenide** - автоматизация браузерных тестов (заполнение форм, клики)
4. **JUnit 5** - организация тестовых классов, запуск сценариев, проверка ожидаемых результатов
5. **Docker** - позволяет запускать приложения и сервисы в изолированных контейнерах
6. **JavaFaker** - генерирует реалистичные фиктивные данные
7. **Lombok** - фреймворк для автогенерации кода с целью улучшения читаемости тестов
8. **Allure** - генерирует наглядные отчеты после прогона тестов, упрощает анализ результатов
9. **GitHub** - средство для контроля версий проекта и сохранения его изменений
10. **Google Chrome** - совместим с Selenide. Необходим для выполнения браузерных тестов
11. **DBeaver** - универсальный SQL‑клиент с поддержкой множества СУБД

## Перечень и описание возможных рисков при автоматизации ##

1. **Падение автотестов после обновлений**. Тесты могут падать из‑за изменений в интерфейсе, логике
2. **Сложности настроить окружение для тестов**. На настройку может уйти много времени, прежде чем тесты начнут работать
3. **Не хватает знаний**. Недостаток знаний по настройке инструментов автоматизации, анализу логов падений
4. **Ресурсные ограничения**. Низкая производительность ПК ведет к медленному выполнению тестов
