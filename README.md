# Руководство по запуску автотестов #

## Предварительные требования ##
- Java 11 - платформа для запуска приложения и тестов
- IntelliJ IDEA - IDE для открытия проекта 
- Docker - для запуска СУБД в контейнере
- Git - для клонирования репозитория
- Chrome - для просмотра интерфейса веб‑приложения

## Шаги для запуска тестов ##
1. Клонирование репозитория
- Выполните команду в терминале git clone https://github.com/vicfadeewa/summary-qamid-q118

2. Открытие проекта в IntelliJ IDEA
- Запустите IntelliJ IDEA
- Выберите File → Open и укажите путь к каталогу проекта (summary-qamid-q118)

2. Запуск СУБД MySQL в Docker
- Открыть программу Docker
- Убедитесь, что Docker запущен (внизу статус Engine running)
- Запустить контейнеры базы данных командой в терминале IntelliJ IDEA docker compose up

3. Запуск приложения
- Запустить приложение командой в терминале IntelliJ IDEA java -jar ./artifacts/aqa-shop.jar
- Проверьте доступность приложения, введя в браузере ссылку http://localhost:8080 
- Должна открыться страница "Путешествие дня"

4. Запуск автотестов и генерация отчетов Allure
- В терминале IntelliJ IDEA ввести команду ./gradlew clean test
- В терминале IntelliJ IDEA ввести команду ./gradlew allureServe
